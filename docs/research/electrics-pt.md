# 07-bis · Нормы: методология расчёта — Электрика в ремонте (Португалия)

> Глава дизайн-дока, продолжение методологии гл.07. Источник: 6 research-досье (cable-norms, panel-circuits, rtiebt-minimums, certification-legal, prices, process-sequence) по электрике жилого ремонта Лиссабона. Регуляторная база — **RTIEBT** (Regras Técnicas das Instalações Eléctricas de Baixa Tensão, Portaria 949-A/2006), разд. **801** (жильё) и **701** (ванные). Формат — как в гл.07: метод → входы → диапазон/seed → источник → confidence. Все спорные числа помечены; где досье противоречат — указан конфликт и выбор.

## 0. Центральный тезис (наследует тезис гл.07, но инвертирует его)

Гл.07 (плитка/стяжка/краска): расход **не выводим из габаритов**, главная неопределённость — физика слоя (ровность, калибр, впитываемость) → Risk #1 = калибровка.

Электрика **инвертирует профиль неопределённости**. Здесь количество дискретных позиций (розетки, выключатели, светильники, цепи, модули щита) считается **детерминированно от числа точек и типа квартиры** — это сильная сторона. Зато **метраж кабеля/гофры на точку a priori неизвестен** (зависит от трассировки по плану, высот, положения щита) — это и есть Risk #1 электрики, прямой аналог «толщины стяжки по кривизне».

Поэтому архитектурное правило то же, что в гл.07: движок отдаёт **формула(детерминированные входы) → диапазон + флаг калибровки**, а не точку. Но граница «детерминированное / калибруемое» проходит **по другому материалу**: считаемое-точно = штучные позиции; калибруемое = погонаж (кабель, гофра, труба).

Две дисциплины, которые нельзя путать (как в гл.07 Слой1/Слой2):
- **Слой 1 — обмер штук (детерминированно):** N точек → N коробок, N механизмов; N точек → доля цепей; тип квартиры → модули щита. Прямой счёт, диапазон узкий.
- **Слой 2 — погонаж на штуку (калибруемо):** м кабеля/точку, м гофры/точку. **Нормы «метры на точку» в RTIEBT НЕТ** [cable-norms, confidence: high на отсутствие]; число — обиходная эвристика, калибруется по факту.

Важно для scope (см. §6): **MVP Obrato = `electric_points` без трасс** (гл.05 §WorkSelection: «MVP = позиции без трасс»). Это значит, что детерминированный Слой-1 движок может закрыть сейчас, а погонаж (Слой-2) — это либо грубый per-point BOM с широким диапазоном + флаг калибровки, либо фаза-2 (трассировка по плану). Это решение §7.

## 1. По каждому блоку: метод, входы, диапазон, seed, источник, confidence

### 1.1 Кабель внутренней проводки (fio H07V-U / -R)

**Метод/формула.** Метраж НЕ из площади квартиры и НЕ из периметра напрямую. Цепочка: `число точек → средняя длина трассы на точку (горизонталь по rodapé/потолку + вертикаль до высоты точки) × число жил цепи + запас`.

```
м_кабеля ≈ Σ(точки) × ср_длина_на_точку_м × жил_в_цепи + распределит.магистрали
ср_длина_на_точку — КАЛИБРУЕМЫЙ параметр (НЕ в норме), запас 10–15%
жил: освещение/розетки = 3 (фаза+N+T); считается на каждую жилу отдельно при закупке по цвету
```

**Тип кабеля (детерминированно по назначению) [high]:** внутри квартиры всегда **H07V-U/-R 450/750 V внутри трубы** (одножильный «fio V», НЕ кабель с оболочкой); вводы/стояки/подземка — **XV/VV/LXV 0,6/1 kV** (многожильный с оболочкой). Для пилота (квартира, тип C ≤6,9 kVA) релевантен почти только H07V в гофре. Источник: rolearmais.pt, elandcables.com, реальный PT-проект projetosengenharia.

**Сечение по цепи (детерминированно, RTIEBT 801.5.x) [high]:** освещение **1,5 мм²** / розетки **2,5 мм²** / плита **4 мм²** (мин.) / тяжёлые (бойлер, AC) до **6 мм²**. Подтверждено тремя досье (cable-norms, panel-circuits, rtiebt-minimums) перекрёстно.

**Входные переменные:** число точек (детерм.), сечение (детерм. по типу цепи), **ср_длина_на_точку (КАЛИБРУЕМО, Risk #1)**, число жил, запас%.

**Диапазон ср_длины на точку:** **6–10 м/точку** — обиходная эвристика PT-электриков, **первоисточником НЕ подтверждена** [cable-norms, confidence: low]. Плотность периметра (1 розетка / 5 м периметра комнаты, кухня 1 / 3,5 м) — это **бразильская NBR 5410**, в PT используется лишь как ориентир, **не PT-норма** [cable-norms, medium — помечено как BR].

**Seed:** ср_длина **8 [6–10] м/точку** на жильную трассу, запас **+12% [10–15]**, confidence: **low / unvalidated** — это эвристика, не норма; флаг «калибровать по факту» обязателен.

**Источники:** [cable-norms]: RTIEBT 801.5; реальный PT-проект (projetosengenharia.com, Memória Descritiva — самый ценный первоисточник, удельная мощность 25 VA/m²); [Hortavolt дименсионирование](https://hortavolt.pt/dimensionamento-de-condutores-eletricos-segundo-a-rtiebt-guia-tecnico-completo-com-casos-reais/).

**Confidence: тип/сечение — high; метраж/точку — low.** Это «толщина стяжки» электрики.

> **Конфликт между досье:** [prices] прямо отмечает грубую ошибку веб-поиска — «рулон 100 м = 30,99 €» был ошибочно прочитан как «30,99 €/м». Реальная розница H07V-U 2.5: **~0,31 €/м рулоном (LM) vs 0,47 €/м поштучно (watt.pt)**. **Выбрал:** для seed-ориентира беру рулонную цену (закупка ремонта идёт рулонами 100 м), поштучный метр держу как верхнюю границу.

### 1.2 Гофра / труба (corrugado / tubo VD)

**Метод/формула.** Геометрически = длина кабельных трасс (одна труба несёт несколько жил одной цепи), т.е. ≈ метраж кабеля / число_жил_в_трубе + вертикали. На практике для грубого BOM привязывается к точкам.

**Номенклатура (детерм.) [high]:** **corrugado/anelado** (гибкая, рулоны 25/50/100 м, Ø16/20/25) — для замоноличивания в стены/потолок (основной случай ремонта); **tubo VD** (жёсткая ПВХ, палки **по 3 м**, Ø16 освещение/розетки, Ø20 плита/мощные, Ø32 ввод квартиры, Ø50/63 стояки); **ERFE/isogris** — в полу. Диаметр трубы — по таблице RTIEBT от числа и сечения жил. Источник: [cable-norms] PDF стр.20–23, Leroy/Obramat; продаётся **не в погонных метрах вразнобой, а рулонами/палками** — важно для pack_size.

**Входные переменные:** длина трасс (= калибруемо, как кабель), Ø (детерм. по цепи).

**Диапазон/seed:** ≈ **0,9–1,1 × метража кабельных трасс** (грубо 1 м гофры на ~1 м трассы, т.к. жилы цепи идут в одной трубе). Seed: **≈ длина трасс × 1.0**, confidence: low (наследует неопределённость метража).

**Источник:** [cable-norms] §5; [process-sequence] §3 (трубы по NP-1702, замоноличены раствором ≥1,5 см).

**Confidence: low на метраж (тот же Risk #1, что кабель), high на номенклатуру/Ø.**

### 1.3 Коробки (caixas)

**Метод/формула.** Детерминированно от числа точек.

```
caixas_aparelhagem = N точек (1 коробка под механизм на каждую розетку/выключатель/вывод)
caixas_derivação   ≈ 1 на разветвление цепи в комнате ≈ грубо 1 на (комната × цепь)
```

**Размеры (детерм., RTIEBT-проект) [high]:** aparelhagem круглая ~**69×44 мм**; derivação до 5 вводов **80×80**, >5 вводов **120×80**; passagem 40×80. Источник: [cable-norms] §6 PDF стр.23, [process-sequence] §5 (Leroy caixa redonda 69×44).

**Входные переменные:** N точек (детерм.), число разветвлений (полу-детерм.: оценка по комнатам×цепям).

**Диапазон/seed:** **1 caixa_aparelhagem / точку** [confidence: medium — это монтажная логика, не норма RTIEBT]; derivação: **~1 / (комната × активная цепь)** [low — эвристика]. Seed: aparelhagem = N_точек × 1.0 (узкий диапазон 1.0–1.05 на бой); derivação — грубо, флаг калибровки.

**Источник/confidence:** [cable-norms] §6 — явно: «точного норматива коробок на точку нет» → medium/low.

### 1.4 Механизмы (mecanismos: tomadas / interruptores)

**Метод/формула.** Полностью детерминированно: `механизмы = N точек` (1 розетка/выключатель = 1 механизм + суппорт + доля рамки). Это самый твёрдый блок.

**Бренд-дефолт PT [medium]:** **Efapel Logus 90** — доминирующий бытовой бренд; бюджет — Schneider Asfora; премиум — с USB/стекло. Источник: [prices] §6, [process-sequence] §5.

**Входные переменные:** N точек (детерм.), тип финиша (драйвит только цену, не количество).

**Диапазон/seed:** количество = N точек, **диапазон 1.0–1.0 (fixed)**. Цена «собранной точки» (механизм+суппорт+рамка): бюджет Asfora **~2–4 €**, Efapel Logus 90 в сборе **~5–9 €**, дизайнерский **13–25+ €** [prices, medium].

**Confidence: high на количество (детерминированно), medium на цену.**

### 1.5 Щит и защита (quadro: disjuntores + diferenciais + DG + DPS)

**Метод/формула.** Детерминированно от типа квартиры (T1/T2/T3) через число цепей → число модулей.

```
модули_щита ≈ DG(2) + DPS(2–4) + 2×RCD(4) + Σ disjuntores(1 каждый) + резерв(20–30%)
число disjuntores ≈ число активных цепей (см. §1.6)
типоразмер щита: ряды 8/12/18/24/36/48/72 мод
```

**Состав (детерм., RTIEBT/практика) [high]:** вводной DG 2P 40A; DPS/SPD Type2 (обязательность зависит от анализа риска — [panel-circuits] medium); **≥2 диф.реле RCD 2P 40A 30mA** (влажные/уличные цепи обязательно под 30 mA — RTIEBT 701/801); disjuntores MCB 10/16/20/25 A по цепи. Резерв **≥20–30%** свободных модулей. Источник: [panel-circuits] §1,5; Hortavolt, Schneider Resi9, Efapel-каталог.

**Входные переменные:** тип квартиры (детерм.), число цепей (полу-детерм., §1.6).

**Диапазон/seed по типам [panel-circuits, high для T2/T3, medium для T1]:**

| Тип | модулей щита | занято модулей | disjuntores |
|---|---|---|---|
| T1 | **24** | 14–18 | ~6–9 |
| T2 | **24–36** | 20–26 | ~9–13 |
| T3 | **36** | 26–30 | ~12–16 |

Опорный «золотой» пример (T2/T3 monofásico, Hortavolt): 2 свет + 3 розетки + 2 кухня + стиралка + посудомойка + бойлер + AC + 2 резерва = **13 активных цепей, щит 36 мод** [high].

**Источник:** [panel-circuits] §4,5; [Hortavolt щит](https://hortavolt.pt/como-dimensionar-e-escolher-o-quadro-eletrico-certo-guia-tecnico-para-profissionais/); quadrina.pt, fioblu.com (готовые щиты).

**Confidence: high для T2/T3, medium для T1.**

> **Конфликт:** [panel-circuits] прямо отмечает — **жёсткого обязательного минимума числа цепей по T1/T2/T3 в RTIEBT НЕТ**. Норма задаёт «макс 8 точек/цепь» + «раздельность освещение/розетки», а число цепей выводится из площади/оснащения. **Выбрал:** трактовать таблицу выше как «репрезентативную практику/seed», не как императив (как в гл.07 waste% — иерархия твёрдая, точное число trade-practice).

### 1.6 Цепи на T1/T2/T3 (число финальных цепей)

**Метод/формула.** Полу-детерминированно. Жёсткие правила RTIEBT задают каркас, число — из практики.

**Жёсткие правила (RTIEBT, текст) [high]:**
- **Макс. 8 точек использования на финальную цепь розеток** (801.5.3); **двойная/тройная розетка в одном аппарате = 1 точка**. Подтверждено тремя досье + Diário da República. → `circuits_розетки = ceil(N_розеток_точек / 8)`.
- **Освещение и розетки — НИКОГДА в одной цепи** (раздельные канализации).
- **Выделенные цепи** под тяжёлые приборы (каждый свой disjuntor): плита, духовка, бойлер, стиралка, посудомойка, AC.
- Лимит точек на цепь **освещения** RTIEBT'ом **НЕ подтверждён** (PDF не парсятся) — обиходно ~8, эвристика [cable-norms, low].

**Входные переменные:** N розеточных точек (→ цепи розеток детерм. через /8), список тяжёлых приборов (детерм. — каждый = 1 цепь), число свет-зон, число ванных/кухонь.

**Диапазон/seed (см. таблицу §1.5):** T1 ~6–9, T2 ~9–13, T3 ~12–16 активных цепей.

**Источник:** [panel-circuits] §3,4; [rtiebt-minimums] §2; [analitica.pt](https://analitica.pt/perguntas-frequentes/), Diário da República 2006-70055500.

**Confidence: high на «8 точек/цепь» и раздельность; medium на итоговое число цепей.**

## 3. Регламент PT — обязательные минимумы (RTIEBT)

**Сечения по цепи [high, текст RTIEBT 801.5.x]:** освещение **1,5/10A**, розетки **2,5/16A**, плита **4 мм²** (мин.; практика 6 мм²/32A — это практика, не норма [rtiebt-minimums]§3), бойлер/машины **2,5 мм²**.

**УЗО/дифзащита [high]:** все цепи розеток ≤30 mA; **все цепи ванной ≤30 mA — RTIEBT 701.411.1.4.3**; наружные ≤30 mA. Освещение часто 300 mA как головной. Архитектура: **≥2 RCD**, влажные/уличные — обязательно под отдельный 30 mA.

**Ванная — зоны/volumes (RTIEBT 701) [medium]:** V0 только TRS ≤12V; V1 над ванной до **~2,25 м** (только TRS); V2 периметр **0,60 м** до 2,25 м (свет/полотенцесушитель, розеток нет кроме класса II); V3 — розетки разрешены с ≤30 mA. **IP по объёмам [high, дословно 701.512.2]: V0=IPX7, V1=IPX5, V2=IPX4, V3=IPX1.** Уравнивание потенциалов 2,5/4 мм² [high].

> **Конфликт по высотам зон:** источники дают 2,25 м vs 2,5 м vs 3 м. **Выбрал 2,25 м** — каноническая граница HD 60364-7-701, на которой основан RTIEBT [rtiebt-minimums]§5 явно так рекомендует; пометка confidence: medium, сверить с официальным PDF 701.

**Кухня [high практика]:** выделенные цепи под плиту/духовку/посудомойку/стиралку; минимум ~6 розеток (рыночный baseline). RTIEBT не даёт закрытого перечня — это устойчивая практика.

> **Критичный методологический флаг [rtiebt-minimums, high]:** **RTIEBT раздел 801 НЕ задаёт прескриптивную таблицу «N розеток на комнату X»** (в отличие от испанского REBT ITC-BT-25). Цифры розеток по комнатам (sala 4 / cozinha 6 / quarto 3 / WC 1) взяты из рыночных источников (idealista.pt, maisconsultores.pt — совпадают), это **«baseline/ожидание клиента», НЕ закон**. Obrato должен предзаполнять их как шаблон (`RoomTemplate.default_works`), но не выдавать как нормативный минимум.

**Легальность [certification-legal, high — для дисклеймера UI]:** Obrato как калькулятор лицензии не требует. Целевая зона пилота — **тип C НТ, ≤6,9 kVA, monofásico**: проект НЕ нужен, начальная инспекция НЕ нужна, достаточно `termo de responsabilidade pela execução` + `ficha eletrotécnica` (~50 €) от техника, зарегистрированного в DGEG. **Триггеры усложнения, которые Obrato должен флагить:** мощность >10,35 kVA или трёхфаза → нужен projetista + проект (~750 €). **Терминологическая ловушка:** «сертификат CERTIEL» устарел с 01.01.2018 → правильно ficha eletrotécnica + termo + инспекция EIIEL раз в 5 лет (жильё ≤6,9 kVA освобождено).

## 5. Процесс — стык с этапами Obrato

Электрика ложится на 2 этапа канонического enum (гл.05 §5 `Stage`): **`rough_plumbing_electric`** (черновая) и **`final`** (чистовая). PT-терминология строго делит: **tubagem/negativos** (rough-in) vs **acabamento**.

| Этап (гл.05) | Электрика | Материалы (Слой) | Источник |
|---|---|---|---|
| demolition | снос старой проводки/щита | — | [process]§2 |
| **rough_plumbing_electric** | roços → tubo/corrugado → caixas → enfiamento кабеля → корпус щита; **+ ITED** (телеком, те же черновые) | **кабель, гофра/труба, коробки, корпус щита** (Слой-2 калибруемый + Слой-1 коробки) | [process]§2 high |
| screed | трубы в полу — **точка невозврата** (до заливки betonilha) | (трубы уже уложены) | [process]§2 high |
| waterproofing | электрика мокрых зон уже в трубах; после — не штробить | — | [process]§2 low (прям. источника нет) |
| tiling | коробки выставлены по уровню финиша; розеток ещё нет | — | high |
| wall_ceiling_finish | заполнение штроб раствором 1:3 без извести, штукатурка, покраска | — | high |
| flooring | замоноличено | — | high |
| **final** | mecanismos в коробки + сборка щита (disjuntores+RCD) + тесты + ficha+termo → ligação E-Redes | **механизмы, аппараты щита (MCB/RCD/DG/DPS), щит в сборе** (Слой-1 детерм.) | [process]§2,4 high |

**Главное правило тайминга [high]:** трубы, замоноличиваемые в штробы, **проверяются ДО заполнения раствором**; покраска/финиш — в самом конце (электрик вернётся сверлить). **Лиссабон-специфика [medium]:** шумные работы (штробление) только будни 08:00–20:00; запрещено штробить несущие (pilares/vigas).

**Импликация для модели:** `rough_plumbing_electric` несёт **материалоёмкий калибруемый погонаж** (кабель/гофра — Risk#1), `final` несёт **детерминированные штучные** позиции (механизмы/аппараты). Это удачно ложится на split §0: разные этапы — разный профиль неопределённости.

## 6. Маппинг на модель данных (Material / Sku / NormRule — гл.05 §3,§4)

Предлагаемые **новые Material.key** (формат как в гл.10 §2 — key/category/base_unit/variability):

| `key` | category* | base_unit | variability | этап | seed-pack пример |
|---|---|---|---|---|---|
| electric-cable | wiring | **m** | **high_variance** | rough_plumbing_electric | fio V 2,5 рулон 100м |
| electric-conduit | conduit | **m** | high_variance | rough_plumbing_electric | corrugado Ø20 рулон 100м |
| electric-box | box | **pcs** | ranged | rough_plumbing_electric | caixa aparelhagem 69×44 |
| electric-mechanism | mechanism | **pcs** | fixed | final | Efapel Logus 90 tomada |
| electric-breaker | protection | **pcs** | ranged | final | MCB 16A / RCD 30mA |
| electric-panel | panel | **pcs** | ranged | final | quadro 36 мод |

*category — потребует расширить `Material.category` enum (гл.05 §4) значениями wiring/conduit/box/mechanism/protection/panel; аддитивно.

**NormRule (гл.05 §3) для каждого:**
- `driving_measure` — нужен **новый вариант enum: `electric_point_count`** (число точек) и/или `room_count`/`circuit_count`. Текущий enum (floor_area_m2 и т.д.) электрику не покрывает → расширение обязательно.
- `per_unit: NormValue` — для кабеля/гофры central=8 [6–10] м/точку, confidence=**unvalidated** (честный Risk#1); для механизма/коробки 1.0 [1.0–1.05]; для модулей щита — функция типа квартиры (не от точки → отдельный rule на `driving_measure=apartment_type`, либо预computed по T1/T2/T3).
- `waste_factor: NormValue` — кабель +12% [10–15] (запас на трассировку); штучные ~+2–5% (бой).
- `variability_source` — нужен **новый enum-вариант: `cable_routing`** (трассировка) — прямой аналог `substrate_flatness`. Драйвит «confirm by experience»-UI.
- `assumptions: NormAssumptions` — нужны **новые поля:** `cable_length_per_point_m?`, `apartment_type?{T1,T2,T3}`, `circuit_count?` — структурная замена free-text (как trowel_notch_mm для клея).

**Sku.pack_size инвариант (гл.05 §4, гл.10 §2):** кабель/гофра base_unit=m, pack_size=100 (рулон) → ceil(raw/100) рулонов; механизм/аппарат base_unit=pcs, pack_size=1 (или N для паков, напр. Asfora pack 5 → pack_size=5). Инвариант `pack_unit==base_unit` соблюдается без `coverage_per_pack`.

**`WorkSelection.electric_points: object{sockets, switches, lights}` (гл.05 §WorkKind) — это и есть детерминированный вход Слоя-1:** N точек = sockets+switches+lights, предзаполняется по RoomType (sala/cozinha/quarto/WC — таблица §3 как `RoomTemplate.default_works`). MVP «позиции без трасс» (гл.05) = ровно этот объект; трассировка (фаза 2) добавит per-trace геометрию.

## 7. Practical-вывод: грубый per-point BOM — ДА, как ОРИЕНТИР с флагом калибровки

**Возможен ли грубый per-point BOM?** Да — и он естественно расщепляется на твёрдое и мягкое, как Слой-1/Слой-2 гл.07:

```
ДЕТЕРМИНИРОВАННО (узкий диапазон, confidence high/medium) — считается СЕЙЧАС, MVP:
  механизмы   = N точек × 1.0
  коробки     = N точек × 1.0 (aparelhagem)
  цепи розеток= ceil(N_розеток / 8)            ← RTIEBT 801.5.3, твёрдо
  активные цепи, модули щита, RCD = f(T1/T2/T3) ← таблица §1.5, репрезентативно
  RCD 30mA ванной/кухни, сечения по цепи        ← RTIEBT, твёрдо

КАЛИБРУЕМО (широкий диапазон + ФЛАГ «уточнить по факту», confidence low) — Risk#1:
  кабель  = N точек × 8 [6–10] м × жилы + 12%   ← эвристика, НЕ норма
  гофра   ≈ метраж трасс × 1.0                   ← наследует
  caixa derivação ≈ комнаты × цепи               ← монтажная логика
```

**Как это ложится в модель (D7-механика гл.05/гл.07):**
- Штучные позиции → `NormValue` с lo≈central≈hi, `variability=fixed/ranged`, confidence medium. Движок выдаёт почти-точку.
- Кабель/гофра → `NormValue` с hi >> central, `variability=high_variance`, `confidence=unvalidated`, `variability_source=[cable_routing]`. Движок выдаёт **диапазон + «confirm by experience»**, ровно как стяжка по кривизне (гл.07 §3). Калибровка по `ProcurementActual.consumed_quantity` (реальный метраж купленного кабеля) → `Reconciliation` атрибутирует ошибку на `per_unit` (ср_длина/точку), не на геометрию.
- Композиция диапазона — та же документированная функция ядра (гл.05 §8): `quantity.central = N_точек × per_unit.central × (1+waste)`; границы — per_unit несёт неопределённость, waste полосой поверх.

**Вывод одной строкой:** электрика — **зеркало гл.07**. Там габариты есть, физика слоя неизвестна; здесь физика штук известна, трассировка погонажа неизвестна. Грубый per-point BOM честен **только если кабель/гофра отдаются диапазоном с флагом калибровки**, а штучное — почти-точкой. Это не баг, это Risk#1, встроенный как фича — тот же контракт, что весь движок Obrato.

## 8. Что НЕ подтверждено / расхождения (для DECIDE-секции)

1. **Метраж кабеля на точку (6–10 м)** — обиходная эвристика, **первоисточником не подтверждена** [cable-norms, low]. Главный кандидат на раннюю калибровку по факту Давида.
2. **Коэффициент «м кабеля на м² квартиры»** — в PT-источниках **отсутствует**; считать только через точки.
3. **Лимит точек на цепь освещения** — RTIEBT не извлечён (PDF не парсятся); подтверждён только лимит розеток (8) [cable-norms/panel-circuits].
4. **Жёсткий минимум числа цепей по T1/T2/T3** — в RTIEBT **отсутствует как императив**; таблица §1.5 = практика [panel-circuits, явно].
5. **Таблица розеток по комнатам в RTIEBT 801** — **вероятно не существует** в прескриптивном виде; цифры рыночные [rtiebt-minimums, high на отсутствие].
6. **Высоты зон ванной** — конфликт 2,25/2,5/3 м; выбрано 2,25 м (HD 60364-7-701), сверить с офиц. PDF 701.
7. **Цены Schneider Easy9/Resi9, Legrand** — за B2B-логином, не подтверждены публично [prices, low].
8. **Обязательность DPS/SPD для квартиры** — зависит от анализа риска, не «всегда» [panel-circuits, medium].
9. **Стык электрики с гидроизоляцией** — выведен логически, прямого PT-источника нет [process, low].

**Самый ценный первоисточник:** реальный PT-проект Memória Descritiva (projetosengenharia.com) — сечения, трубы, защиты, 25 VA/m² «как в жизни». **Ключевые нормативные:** Diário da República 2006-70055500 (RTIEBT консолидир.), analitica.pt (цитирует номера статей 801/701), DL 96/2017 (типы A/B/C, пороги проекта).

## Адверсариальные замечания ревьюера (critic-агент)

Адверсариальный ревью отчёта по электрике (PT). Прошёлся по числам, источникам и сверил отчёт с сырьём-досье.

## ⚠ Исправить

1. **Сечение/защита плиты — отчёт нормирует «4 мм²» как минимум, но защиту даёт противоречиво.** §1.1/§3 пишет «плита 4 мм² (мин.)», но в §3 «розетки 2,5/16A, плита 4 мм²» без автомата, а сырьё [rtiebt-minimums]§3 даёт для плиты явно **4 мм² + 25 A**, [cable-norms] — **4mm²/20A**, [panel-circuits] — **20 A**. Три досье дают три разных автомата на 4 мм² (20/20/25 A) — отчёт это расхождение проглотил, выдав «4 мм² (мин.)» без номинала. Для BOM это важно (disjuntor — штучная позиция). Нужно пометить как конфликт.

2. **«ceil(N_розеток / 8)» — формула искажает норму.** RTIEBT 801.5.3 (по всем трём досье) = «макс 8 **точек использования** на цепь», причём точки — это не только розетки, а pontos de utilização (включая выводы). Отчёт сам в §1.6 пишет «розеточных точек», но формула и таблица §2 («Цепи розеток ceil(N_роз/8), Детерм (RTIEBT), high») подаются как твёрдо-нормативные. Лимит именно по розеткам в досье нигде не оговорён отдельно — это интерпретация. Confidence «high» на формулу завышен.

3. **Удельная мощность «25 VA/m²» подана как универсальная константа.** §2 и §8 дают «25 VA/m², high». Сырьё [cable-norms]§4 уточняет: 25 VA/m² — это **освещение+розетки общего пользования**, а 10 VA/m² гаражи и т.д. — то есть это компонент, не полная нагрузка квартиры. В отчёте подаётся как готовый коэффициент для kVA без оговорки про состав → риск недосчёта мощности.

4. **Цена кабеля 1,5 мм² «~0,31 €/м, medium» противоречит сырью.** §4 отчёта: 1,5 = 0,31 €/м. Но [cable-norms]§7 даёт 1,5 = **0,24 (CHL) / ~0,34 (Sapol)**, [prices]§4 — рулон LM 1,5 ≈ 30,99 €/100м = 0,31 (но тут же [process-sequence]§5 даёт рулон 1,5 = **26,29 €/100 м = 0,26 €/м**). То есть на рулон 1,5 мм² два разных числа в сырье (26,29 vs 30,99) — отчёт взял одно молча. Мелочь, но это именно «число без честного источника».

5. **«6 мм² = 1,26 €/м, high» — это цена для 6 мм², а 30,99 €/100м рулон относится к 2,5, не к 6.** Проверь таблицу §4: строки внутренне ок, но confidence «high» на 6/10 мм² держится на одном зеркале (watt.pt) — в [cable-norms] 6 мм² = 1,27–1,61 (разброс 27%). Для high нужен консенсус; здесь скорее medium.

6. **Высоты зон ванной: выбор «2,25 м» обоснован ссылкой «[rtiebt-minimums]§5 явно так рекомендует».** Сырьё [rtiebt-minimums]§5 говорит мягче: 2,25 м — «наиболее вероятное» число по HD 60364-7-701, confidence **medium**, и прямо просит сверить с офиц. PDF 701. Отчёт §3 повышает это до «выбрал 2,25 м» с видом решённого, хотя в §8 честно оставляет флаг. Внутреннее противоречие тона: в §3 звучит твёрже, чем позволяет источник.

7. **Caixa derivação «~1 / (комната × цепь)» — эвристика, выданная с псевдоформулой.** §1.3 и §2 дают диапазон 0.5–1.5 и «полу-детерм». Сырьё [cable-norms]§6 прямо: «точного норматива нет, монтажная логика, [low]». Для per-point BOM это значимая штучная позиция (коробки — деньги), а отчёт даёт формулу комнаты×цепи без указания, что число разветвлений в принципе не выводится из числа точек без плана. Это скрытый второй Risk (не только погонаж калибруем).

8. **Ставка электрика «30–50 €/ч» в §4 vs сырьё.** [prices]§3 даёт CYPE 28+27 €/ч и «рыночно 25–50, Лиссабон верх»; [certification-legal]§8 — «~30 €/ч, диапазон 10–40, Лиссабон верх, не подтверждено отдельно [low/medium]». Отчёт схлопнул в «30–50 €/ч» — это верхняя интерпретация, нижняя граница рынка (10–15) потеряна. Не критично (работа не в BOM), но число подано увереннее источника.

## ➕ Дозакрыть

1. **Жилы на цепь и расчёт по цвету.** Формула кабеля §1.1 умножает «× жил в цепи = 3 (фаза+N+T)», но для per-point BOM закупка идёт **по цвету/по жиле отдельно** (одножильный fio V). Не закрыто: сколько метров КАЖДОГО цвета (фаза/нейтраль/земля) — а это влияет на pack (рулон = один цвет). Сейчас «3 жилы» схлопнуто в общий метраж.

2. **Сечение → разные Material.key или один с вариантом?** §6 даёт один `electric-cable`, но сечений минимум 4 ходовых (1,5/2,5/4/6) с разной ценой (0,31 → 1,26 €/м, разница 4×). Per-point BOM должен раскладывать кабель по сечению цепи — это или 4 SKU, или driving-параметр. Маппинг §6 это не покрывает: `per_unit` для кабеля один на все цепи.

3. **ITED (телеком) полностью выпал из BOM-вывода.** Сырьё [process-sequence]§6 явно: ITED кладётся в тот же rough-этап (труба VD + UTP/coax/fibra + ATI). Отчёт упоминает ITED одной строкой в §5, но в §6 (Material.key) и §7 (per-point BOM) его НЕТ. Для решения Obrato про per-point BOM это пробел: телеком-точки — отдельные точки с отдельным материалом.

4. **Заземление / уравнивание потенциалов как материал.** Сырьё ([rtiebt-minimums]§5, [cable-norms]§2) даёт обязательную ligação equipotencial 2,5/4 мм² в ванной + земляная жила 35 мм² на дом. В per-point BOM и Material.key (§6) земли/уравнивания нет вовсе — а это обязательная по RTIEBT штучно-погонная позиция для мокрых зон.

5. **Кол-во точек по типу квартиры (T1/T2/T3) не сведено с числом цепей.** §1.5/§1.6 дают цепи по типам, §3 — розетки по комнатам, но нет моста «T2 = сколько комнат каждого типа = сколько суммарно точек». Для детерминированного Слоя-1 («N точек → всё остальное») нужен дефолтный состав комнат на T1/T2/T3 — иначе вход N точек не предзаполняется из типа квартиры, на чём держится весь §7.

6. **pack_size для палок tubo VD (3 м) vs рулонов corrugado (100 м) не разведён.** §6 даёт `electric-conduit` один с pack=100 (рулон). Но жёсткая VD продаётся палками по 3 м ([cable-norms]§5), corrugado — рулонами 100 м. Это два разных pack_size/Sku при одном Material.key — инвариант ceil(raw/pack) даст грубую ошибку округления на коротких трассах VD.

7. **DPS/SPD: ставить в seed-щит или нет — не решено.** §1.5 включает DPS «2–4 модуля» в состав щита, но §8 и сырьё ([panel-circuits]§1,7) говорят «обязательность зависит от анализа риска, medium». Для штучного BOM щита (модули = деньги и типоразмер) нужно бинарное решение seed: включаем DPS по умолчанию или флагим опцией. Отчёт оставил в составе, противореча собственному флагу неопределённости.

8. **Старая проводка под снос (demolition) — нулевой вывод по материалу/объёму.** §5 упоминает «снос старой проводки/щита», но это влияет на закупку (новый полный комплект vs частичная замена). Для per-point BOM не задан вопрос: новостройка-логика (все точки новые) vs ремонт (часть трасс остаётся) — а это меняет метраж в разы.

#### B.1 ➕ Кабель по СЕЧЕНИЮ + ЦВЕТУ: как раскладывать метраж и SKU

Терминологическая опора: внутри квартиры тянут **fio** (одножильный провод, monocondutor sem bainha), НЕ кабель с оболочкой. **Каждая жила цепи — отдельный провод своего цвета**, закупка идёт **рулонами = 100 м, один цвет, одно сечение** [high]. Это ключ ко всей логике BOM.

**Цветовой код (Portaria 949-A/2006 / RTIEBT / IEC 60445) [high]:** фаза (monofásico) = **castanho**; нейтраль = **azul**; земля/PE = **verde-amarelo** (только эта комбинация). Трёхфаза (редко в быту): L1/L2/L3 = castanho/preto/cinzento. В монофазной квартире на цепь идут ровно **3 цвета: castanho + azul + verde-amarelo**.

**Тип кабеля переключается по сечению [high]:** **H07V-U** (сплошной, Class 1) для **1,5 / 2,5 / 4 мм²**; **H07V-R** (многопроволочный, Class 2) для **≥ 6 мм²** (6/10/16). Оба 450/750 V, EN 50525-2-31. SKU-маркировка должна переключаться с H07V-U на H07V-R начиная с 6 мм² (Obramat так и делит). Источник: rolearmais.pt, eficilux, Obramat [cable-section-colour §2].

**Разложение по сечению цепи (RTIEBT 801.5 — минимумы habitação):**

| Цепь | Сечение | Защита | Тип | Conf |
|---|---|---|---|---|
| Iluminação | 1,5 мм² | 10 A | H07V-U | high |
| Tomadas (uso geral) | 2,5 мм² | 16 A | H07V-U | high |
| Máq. lavar / loiça (выдел.) | 2,5 мм² | 16 A | H07V-U | high |

Развилка 4 vs 6 мм² для плиты — **проектное решение** (мощность placa, длина, падение ≤3%), НЕ жёсткий норматив; RTIEBT-минимум = 4 [confidence: high, что это развилка; сам выбор не подтверждён без проекта → флаг «калибровать по факту»].

**Алгоритм метража по (сечение × цвет) — рулонная логика:**
1. Сгруппировать все цепи **по сечению** (1,5 / 2,5 / 4 / 6).
2. Внутри сечения просуммировать общий метраж трасс → базовая длина `L_S`.
3. Заказ **по цветам**: для розеточных/силовых цепей чистый **1:1:1** → castanho ≈ L_S, azul ≈ L_S, verde-amarelo ≈ L_S. Для **освещения** добавить **+10–20 % к фазным цветам** (castanho/preto) на возвратные жилы комутации (interruptor/comutador), нейтраль до выключателя не доходит → расход фазных > azul [confidence: medium — коэф. эвристический, НЕ норматив, флаг калибровки].
4. Запас на отходы/заводку в коробки **+10–15 %** к L_S [medium — практика].
5. **Округлить КАЖДЫЙ цвет вверх до кратного 100 м (рулон)** — главный источник «лишнего» метража, заложить в смету.

**Готовая разбивка «цепь → жилы × сечение × цвет» (монофаза):** каждая монофазная цепь = **3 провода (3 цвета)**; трёхфазная силовая (редко) = 5 проводов. Цена цвета внутри сечения одинакова (verde-amarelo = castanho = azul) [high]. Источник: [cable-section-colour §1,4,6].

#### B.2 ➕ ITED (телеком) — материалы + точки

ITED — **отдельная подсистема** от силовой электрики, со своими точками, кладётся в **тот же rough-этап** (труба/calha + кабели PC/CC/FO + бокс), до закрытия стен. Первоисточник: **Manual ITED 3.ª ed. (ANACOM)**, обязателен при ремонте, проходящем лицензирование Câmara Municipal [high]. Три обязательные сети: **par de cobre UTP Cat6** (Classe E), **coaxial TCD-C-H / T100** (до 3 ГГц), **fibra ótica OS1/OS2** (2 волокна/квартиру в коллективной части).

**Точки по комнатам (минимум ITED3, новостройка/ремонт ITED3a) [high]:**
- В каждой жилой комнате (sala, quartos, cozinha) — **минимум 1 tomada mista RJ45+coax (PC+CC)** или 2 раздельные на ≤20 см.
- **ZAP** (1 на квартиру, обычно sala): **2×RJ45 + 2×coax + резерв под 2×FO** (труба Ø≥25 мм под FO).
- НЕ обязательны: kitchenettes, ванные, прихожие, кладовые, балконы, комнаты <6 м². FO-розетки не обязательны (зарезервировать место).

|---|---|---|---|
| T1 | cozinha 1 + quarto 1 | 2 RJ45 + 2 coax | ~4 RJ45 / 4 coax |
| T2 | cozinha 1 + 2 quartos | 2 RJ45 + 2 coax | ~6 RJ45 / 6 coax |
| T3 | cozinha 1 + 3 quartos | 2 RJ45 + 2 coax | ~7 RJ45 / 7 coax |

**Узловая точка (1 на квартиру):** бокс **PDF** (новостройка) / **PTI+PCS** (ремонт), на рынке = «ATI». Комплект 4 PC + 3 CC + 2 FO, ~360×504×125 мм, IP42/IK08, + эл. розетка + шина земли + место под ONT/router (≥200×250×100 мм). Дорогой штучный элемент.

#### B.3 ➕ Заземление / уравнивание потенциалов — материалы

Схема сети PT-жилья = **TT** → защита от косвенного прикосновения = дифавтомат/IDR 30 мА + локальное уравнивание; формула проверки `RA × IΔn ≤ 50 V` (RTIEBT 413.1.4.2; с 30 мА проходит почти любой электрод). PE и проводники уравнивания — **verde-amarelo** [high].

**Сечения PE (RTIEBT 543.1.2, Quadro 54F) [high]:** S_F ≤16 → S_PE = S_F; 16<S_F≤35 → S_PE = 16 мм²; S_F>35 → S_PE = S_F/2. Проекция: фаза 1,5→PE 1,5; 2,5→2,5; 4→4; 6→6; 10→10; 16→16; 25→16. То есть «16 мм²» как PE появляется лишь на магистрали ввода 25–35 мм². Абсолютный минимум PE вне канализации (543.1.3): **2,5 мм²** (мех. защищён) / **4 мм²** (не защищён).

**Главное уравнивание (547.1.1) [high]:** ≥½ наибольшего PE, минимум **6 мм²**, ограничено сверху 25 мм². В квартире = чаще всего **6 мм² verde-amarelo** от barramento principal de terra к стоякам воды/газа.

**Дополнительное уравнивание ванной (547.1.2 + 701.413.1.6) [high]:** обязательна связь всех проводящих элементов в объёмах 0–3 с PE. Сечение выводится через 547.1.2→543.1.3 (= 2,5 мм² если в трубе / 4 мм² если открыто). **Практика PT (CYPE, Memória): жёсткий медный 4 мм² verde-amarelo** [confidence: high для практики 4 мм²; medium — что 2,5 встречается реже]. Соединяют: трубы воды (гор.+хол.), металлический слив, полотенцесушитель, металлическая ванна/поддон, газ (если есть), PE розеток/светильников. Пластиковые трубы (PEX/PPR) хомута не требуют.

#### B.5 ➕ Решения по щиту (автомат плиты / DPS seed-default / демонтаж)

**(a) Автомат плиты @4 мм²:** см. ⚠1 — **seed 20 A [20–25], сечение 4 мм² fixed, цепь выделенная** [high/medium].

**(b) DPS/SPD — бинарное seed-решение: НЕ включать по умолчанию.**
RTIEBT 801.5.10 (раздел habitação, дословно): DPS — «**recomenda-se**» (рекомендуется, НЕ obrigatório) и **только при воздушном вводе**. RTIEBT 443: подземный ввод → DPS НЕ требуется (443.2.1); воздушный + AQ1 → не требуется (443.2.3). Кераунический порог: **AQ1 = N≤25 дней гроз/год** (DPS не нужен), AQ2 = N>25 → анализ риска. Лиссабон = AQ1 (только Castelo Branco исторически >25) [medium]. В реальном проекте T2/6,9 kVA DPS в щите **отсутствует вовсе** [high как пример].
**Seed: `dps_required = false` (бинарный флаг)** для пилотного типа (квартира, тип C ≤6,9 kVA, Лиссабон) [confidence: high на «не дефолт для Лиссабона»; medium на AQ-классификацию района]. Триггеры включения: (1) воздушный ввод + регион AQ2; (2) здание с молниезащитой (para-raios); (3) явный запрос защиты электроники. Если включён → **DPS Type 2, 1P+N или 2P, ~2 модуля**. Это снимает противоречие первого отчёта (§1.5 включал DPS «2–4 мод» в состав, §8 флагил неопределённость). Источник: [panel-decisions §2], RTIEBT 801.5.10 / 443 / 321.13 (liq.pt).

**(c) Демонтаж: total vs parcial — дефолт `total`, множитель метража.**
Практика PT: при ремонте старого фонда проводку меняют **ЦЕЛИКОМ (rehabilitação total)** — новые roços, новая tubagem И новая cablagem на все точки; старую tubagem НЕ переиспользуют (особенно дома ~1971 г. и старше — не держит нагрузку, сечения не по RTIEBT) [high, forumdacasa 93055, engenheirofaisca]. Частичная замена (провод в существующих трубах) — исключение.
**Seed: `rehabilitation_scope ∈ {total, partial}`, дефолт `total`** [confidence: high на дефолт]. Влияние на метраж: при `total` — полный per-point по всем точкам; при `partial` — **обнулять/резать гофру до ~0** (трубы остаются) и считать кабель только по выбранным цепям/зонам. **Множитель: total vs partial отличаются по полному метражу кабеля в ×2–×4**, total vs «косметика» — до порядка величины [confidence: medium — порядковая оценка, не из единого первоисточника, кандидат на калибровку]. CYPE-калибр (unifamiliar, ~52 точки, 8 цепей, total): 1,5 мм² 648 м; 2,5 мм² 315 м; 4 мм² 54 м; 6 мм² 30 м + труба Ø16 179 м, Ø20 154 м. `rehabilitation_scope` — прямой множитель Risk #1, жить рядом с `cable_length_per_point_m`. Источник: [panel-decisions §3], CYPE IEI015, forumdacasa.

#### C.4 Sku.pack_size — два pack для conduit

Инвариант `pack_unit == base_unit` (m) держится. Под `electric-conduit` — **два Sku**:
- **corrugado** (гибкий, в стены) → `pack_size = 100` (рулон 100 м); `packs = ceil(raw/100)`.
- **tubo VD** (жёсткий, палки) → `pack_size = 3` (палка 3 м); `packs = ceil(raw/3)`.

Без разведения pack `ceil(raw/100)` на коротких VD-трассах даёт грубую ошибку округления (купил бы 100 м вместо 3 палок). Кабель: `pack_size=100` (рулон, 1 цвет). Механизм/аппарат: `pack_size=1` (или N для паков, напр. Asfora pack 5 → 5). Источник: [cable-section-colour], [cable-norms §5].

5. **Предзаполнение точек — НЕ норматив:** дословной RTIEBT-таблицы «N tomadas/комната» не существует; дефолты B.4 = шаблон, в UI не подавать как обязательный минимум.

### 2. Сечение/автомат тяжёлых цепей → seed-дефолты зафиксированы [ФАКТ для минимумов + ⚙ для выбора запаса]

RTIEBT 801.5.8 задаёт минимумы де-юре; запас — инженерная практика PT-форумов. Seed:

| Цепь | Сечение (seed) | Автомат (seed) | Тип | [conf] |
|---|---|---|---|---|
| **Placa индукция (отдельно)** | **6 мм²** | **32 A** (кривая C) | ФАКТ-мин 4/25 + практ. запас | high |
| **Forno (отдельно)** | **4 мм²** | **25 A** | ФАКТ-мин RTIEBT для fogão | high |
| **Placa + Forno (одна цепь)** | **6 мм²** | **32 A** | практ. консенсус | high |
| **Termoacumulador (бойлер)** | **2,5 мм²** | **16 A** | ФАКТ: 1,5–2 кВт = 6,5–8,7 A | high |
| **AC моносплит 9–12k BTU** | **2,5 мм²** | **16 A** (кривая C) | RTIEBT-мин 2,5; кривая C под пуск | medium |

**Резолюция спорных пунктов из остатка:**
- **Плита 4 vs 6 мм²** → seed = **6 мм² / 32 A**. RTIEBT-минимум 4 мм²/25 A — это пол [ФАКТ], но индукция 7,2–7,4 кВт даёт ~32 A, и 4/25 «впритык». 6/32 — устойчивый «золотой стандарт» профи. Кабель 6 мм² кладём всегда; автомат при contratada 6,9 кВА (вводной ~30–32 A) допустимо опустить до 25 A — 32 A иначе бессмысленен (сработает вводной).
- **Бойлер 2,5 vs 4 мм²** → seed = **2,5 мм² / 16 A**. 4 мм² — оверкилл для стандартного 1,5–2 кВт; нужен только при >3 кВт или длинной трассе [medium].

**Обоснование:** eletricidade.net t=8522 / t=16674 / t=19226 (плита, контекст contratada); t=797 + galp.com (бойлер); RTIEBT-минимумы t=17406; таблица silvaluz.pt. [confidence: high для чисел]

🔧 **Флаги:** (а) автомат placa = функция **potência contratada** — при ≥10,35 кВА держим 32 A, при 6,9 кВА опускаем до 25 A (кабель 6 мм² неизменен) — параметризовать по вводу. (б) AC-пороги BTU→автомат для 18k+ частично из BR-источников (NBR 5410), не PT — выше 12k BTU пересчитывать по паспорту блока [low для PT-атрибуции]. (в) Дословная Portaria 949-A/2006 не верифицирована из открытого PDF (платный); минимумы взяты из совпадающей вторички [high для чисел, medium для первоисточника].

### 3. AQ-класс района (триггер DPS) → **вне автоматики MVP; seed = DPS включён по дефолту + ручной флаг** [РЕШЕНИЕ]

**Резолюция (продуктовое решение, НЕ норма):** AQ-класс (вероятность грозового воздействия) **не выводится из габаритов квартиры** — это функция географии/типа здания/внешней линии, которой у движка на входе нет. Поэтому:
- **MVP-стойка:** DPS (descarregador de sobretensões) **предзаполнен в seed как «включён» (AQ ручной)**, с явным **редактируемым флагом** «район/здание требует DPS: да/нет». Дефолт «да» — консервативный (RTIEBT тяготеет к обязательности DPS в новых/реновируемых щитах; дешевле заложить и снять, чем забыть).
- Автовывод AQ-класса из адреса — **вне MVP** (нужен внешний геоисточник риска; не окупается на пилоте).

**Обоснование:** триггер не вычислим из имеющихся входов движка → не может быть «нормой» в проекции; разумный консервативный дефолт + ручной override. [confidence: средняя — это проектный выбор, не факт]

🔧 **Флаг:** значение по умолчанию (вкл/выкл) калибровать по факту первых лиссабонских проектов — если DPS закладывается практически всегда, дефолт «вкл» подтверждён; если часто снимается — переключить дефолт. Кандидат на пост-MVP: таблица AQ по concelho/freguesia.

### 4. Предзаполнение точек (розетки/выводы по помещениям) → **редактируемый шаблон, НЕ обязательный минимум** [РЕШЕНИЕ]

**Резолюция (продуктовое решение):** норматива RTIEBT с таблицей «N розеток на тип помещения» **нет** (в отличие от ITED, где минимум tomadas телеком закреплён). Поэтому числа точек на помещение в движке — это **UX-шаблон, а не нормативный минимум**:
- UI подаёт предзаполнение как **редактируемый default-шаблон** («типовая комплектация», пользователь правит вверх/вниз).
- **Не** маркировать как «минимум по норме» / «обязательно» — это введёт в заблуждение и создаст ложную нормативную ответственность.
- Исключение — **телеком-точки ITED4** (sala/cozinha 2×RJ45+TV, ZAP): **они** нормативны и могут быть помечены как «минимум ITED4» (см. п.1).

**Обоснование:** разделяем два слоя — электрические точки (нет норм-минимума → шаблон) vs телеком ITED4 (есть минимум → норма). Смешивать нельзя. [confidence: высокая для факта «RTIEBT-таблицы нет»; решение по UI — продуктовое]

🔧 **Флаг:** конкретные числа шаблона (напр. розеток на quarto/sala) калибровать по типовым лиссабонским MD и фидбэку электриков-пилотов.

### 5. Демонтаж total/partial → seed + Risk#1-флаг зафиксированы [РЕШЕНИЕ]

**Резолюция (продуктовое решение):**
- **Seed-дефолт демонтажа = `partial`** (частичный — типичный сценарий remodelação: меняется проводка/щит, несущая коробка/штробы частично сохраняются).
- **Множитель сложности total vs partial = ×2–4** к трудозатратам/стоимости демонтажа (total — полный сруб старой инсталляции до бетона).
- **Привязать к Risk#1-флагу:** объём демонтажа — главный источник недооценки сметы; флаг выводит пользователю явный выбор `total | partial` с предупреждением о ×2–4 разбросе и требованием подтверждения перед фиксацией сметы.

**Обоснование:** total/partial — это **проектный выбор заказчика/прораба, не норма**; движок не может определить его из габаритов → дефолт + обязательный явный override. Множитель ×2–4 — оценочная вилка, не измеренный коэффициент. [confidence: средняя — продуктовое решение; сам множитель — оценочный]

🔧 **Флаг (калибровка обязательна):** коридор ×2–4 — **гипотеза**, не замер. Откалибровать по фактическим обмерам первых проектов (часы демонтажа total vs partial на м²/точку). До калибровки в UI показывать как диапазон с дисклеймером, не как точное число.
