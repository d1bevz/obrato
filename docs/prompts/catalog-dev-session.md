# Промпт для сессии: Разработка каталога Obrato

> Открой эту сессию в репо `d1bevz/obrato` (ветка `main`). Скопируй текст ниже как

---

Ты собираешь **каталог материалов** для **Obrato** — web-PWA помощника прораба
(ремонт, Лиссабон). Каталог = курируемый набор товаров, из которого движок берёт SKU
для листа закупок. Пилот N=1 — прораб Давид, рынок Лиссабон/Португалия.

## Сначала прочитай (дизайн-док уже готов)
`docs/design/`:
- `05-data-model.md` **§4 «Каталог»** — сущности **Material / Sku / Store / Price** (точные поля, типы, связи). СТРОГО следовать этой схеме.
- `07-norms-methodology.md` — нормы расхода (каталог + нормы вместе дают количества; у каждого Material есть `base_unit`, в которой нормы считают расход).
- `04-decisions.md` **D10** — **цена = ОРИЕНТИР**, не точная фича: низкая свежесть, integer money (`amount_minor_units`), Price append-only с `captured_at` + `source` + `store`.

## Задача
Собрать **курируемый каталог ~50–100 SKU**, которые Давид реально берёт, в формате
схемы гл.05 §4 (Material → Sku → Store → Price), готовый загрузиться как **глобальный
seed** (`org_id=null`, read-only для клиента).

## Детали (из принятых решений)
- **Материалы** (стартовый набор): floor-tile, wall-tile, tile-adhesive, grout, paint, primer, screed-mix, waterproofing, baseboard + что всплывёт у Давида.
- **Multi-store**: Leroy Merlin PT + локальные магазины Лиссабона. **Open Q2** — какие именно локальные магазины (одного Leroy не хватит, нужны аналоги/наличие). Выяснить, желательно у Давида.
- **Источник данных**: сам `leroymerlin.pt` отдаёт 403 (бот-блок) → бери фабричные fichas брендов, что Leroy PT возит (CIN, Weber, Mapei, Kerakoll, Sika) + **ручная курация по реальным покупкам Давида**. НЕ строить скрейпер (D10 — свежесть некритична).
- **Инвариант пилота**: `Sku.pack_unit == Material.base_unit` (без conversion; редкие исключения — через `coverage_per_pack`, см. гл.05 §4).
- **Связь с нормами**: `Sku.pack_size` в той же `base_unit`, в которой нормы (гл.07) выдают расход — чтобы округление raw → упаковки работало.

## Deliverable
- **Каталог-данные ~50–100 SKU** в формате гл.05 §4 — структурированный seed (JSON или Rust/TS-структуры, как удобнее для загрузки в движок; согласуй с типами гл.05).
- Глава **`docs/design/10-catalog.md`**: как каталог собирается и поддерживается, multi-store-стратегия, решение **Open Q2**, формат seed-данных, процесс обновления цен-ориентиров.
- Закоммить в репо, обнови строку 10 в `00-README.md`.

## Якорь
Опирайся на РЕАЛЬНЫЕ покупки Давида — это и есть валидация (какие SKU реально нужны на объекте), а не теоретический список. Каталог неполон по определению — заложи «off-catalog» путь (см. `Sku.sku_free_text` в гл.05).
