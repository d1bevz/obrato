# 04 · Решения (Decision Log)

Выхлоп CTO/eng-ревью. Каждое: что / почему / что отвергли / статус.

**D1 — Платформа: web-PWA, таргетящая Android.** Принято 2026-05-29.
Android = рантайм/полигон, не среда сборки. Для нагрузки «формы + 2D-вектор + лёгкий
счёт» у веба нет потолка плавности; PWA даёт офлайн на объекте (service worker +
IndexedDB). Отвергнуто: нативка Flutter/Kotlin (низкий ROI для соло-DS, не его стек),
Telegram-бот (нет 2D-плана). Google Play позже — TWA-обёртка поверх PWA.

**D2 — `compute-core` отдельным чистым модулем.** Принято 2026-05-29.
Чистые функции, тесты, ноль зависимостей от UI/DOM. Ядро отдаёт **данные** (геометрия +
количества + лист), Canvas/UI рисует — НЕ SVG-строки из ядра. Один core для PWA (клиент),
headless-PDF и будущих native/server. Зачем: баг в количествах ловится в тестах ядра;
переносимость.

**D3 — Язык ядра: Rust → WASM.** Принято 2026-05-29 (решение Димы).
Почему: строгость/корректность для движка норм (Risk #1) + переносимость одного ядра на
web/native/server без переписывания. Граница: wasm-bindgen + serde-wasm-bindgen; TS-типы
генерятся из Rust (tsify/ts-rs) → нет двойной модели. Отвергнуто: TypeScript. CTO-рекомендация
была TS-now / Rust-when-triggered (ради скорости итерации на ранней фазе); Дима осознанно
выбрал Rust с самого начала ядра, приняв трейд-офф — медленнее итерация через FFI-границу +
кривая обучения Rust. UI остаётся TS/React.

**D4 — UI: React + Vite (vite-plugin-pwa) + Canvas + Web Worker.** Принято 2026-05-29 (решение Димы).
Почему: самый проверенный, крупнейшая экосистема, и — ключевое под «строим с AI» — AI
пишет React увереннее всего (больше всего обучающих данных) → быстрее итерация. Плотную
графику рисуем на Canvas **императивно** (useRef + draw-функция), НЕ тысячами
React-элементов; React держит оболочку (формы, лист, навигация). WASM-ядро считает в
Worker, вне UI-потока. Отвергнуто: Svelte/SolidJS — тоньше рантайм, но перф-критичная
часть (Canvas) фреймворк-независима, так что эта разница для нашей нагрузки маржинальна.

**D5 — Метрика прогресса = валидация норм, не скорость UI.** Принято 2026-05-29.
Прогресс мерим тем, как быстро ядро встретилось с фактической закупкой Давида (Risk #1),
а не полировкой UI. AI ускоряет код, не проверку норм.

**D6 — Нормы: калибровка под тайлера, не только статичная таблица.** Принято 2026-05-29.
Стартовая таблица коэффициентов калибруется по факту закупок Давида (позже —
кросс-тайлерно). Схема **с первого дня** хранит «оценку» и «фактически куплено» рядом —
без этого калибровать нечем. Старт: ручная сверка на пилоте; авто-калибровка
(Bayesian-стиль, как TDEE-трекер) — потом.

**D7 — Высоковариативные материалы: диапазон, не голое число.** Принято 2026-05-29.
Где точность зависит от входов вне Д×Ш×В (стяжка ~ ровность основания, клей ~ зубец
шпателя) — показываем «число ± / подтверди опытом», а не ложно-уверенную цифру.
Честность важнее крутости перед ПРО-юзером.

**D8 — Чертёж: наивная раскладка + ручное редактирование (человек-в-цикле).** Принято 2026-05-29.
MVP не строит идеальную раскладку плитки — даёт черновую + **редактируемый Canvas**, Давид
сам правит датум/подрезку. Решает доверие эксперта дешевле полного солвера (солвер всё ещё
фаза 2). Цена: Canvas интерактивный, не статичный рендер.

**D9 — Бэкенд + multi-tenant модель данных с первого дня; B2B-фичи позже.** Принято 2026-05-29 (решение Димы; сервер у него уже есть).
Схема multi-tenant сразу (`owner/org` на объектах) → нет болезненной миграции single→multi
потом. Но B2B-фичи (роли, админка организаций, отчётность, биллинг) — ПОСЛЕ валидации ядра
на Давиде (он = первый клиент / дизайн-партнёр), чтобы не строить обвязку вокруг
непроверенного ядра + неподтверждённого B2B-спроса. PWA остаётся offline-first (IndexedDB);
синк на пилоте держим простым (N=1, одно устройство), robust multi-device sync — позже.
Усиливает D3: переносимость ядра теперь не «может быть», а день-1 (сервер есть).

**D10 — Цена = ориентир, не точная фича.** Принято 2026-05-29.
В магазине всё равно подтверждается → требование к свежести низкое. Каталог курируется
руками (~50–100 SKU топ-материалов), скрейпер не нужен долго. В листе цена помечена
«ориентир, уточни в магазине».

**Ещё открыто:** стартовый источник коэффициентов норм (Q1 — какие числа берём до
калибровки); локальные магазины помимо Leroy PT (Q2 — какие, и нужны ли на пилоте).
