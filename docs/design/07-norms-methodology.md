# 07 · Нормы: методология расчёта расхода

> Глава дизайн-дока. Источник: 7 research+verify проходов по фичам брендов (CIN, Weber, Mapei, Kerakoll, Sika, Topeca, Robbialac, Dyrup, Isomat) + стандарты (BS 5385, EN 12004/13888/14891, ANSI A108.02, DIN 18534, BS 8204). Все числа прошли verify-перепроверку; где verify нашёл ошибку research — взято исправленное значение и это помечено.

## 0. Центральный тезис методологии

Расход материала **почти никогда не считается из «Д×Ш×В помещения»**. Он считается по цепочке `геометрия работ → физический параметр слоя/шва → плотность/укрывистость материала → масса/объём`, где **минимум один вход в цепочке физически не содержится в габаритах** (ровность основания, ширина шва по калибру партии, впитываемость поверхности, выбор инструмента мастером). Витринное «4 kg/m²» — это уже **свёрнутый итог** этой цепочки для типового случая, и движок, который выдаёт его как детерминированную истину, врёт.

Поэтому архитектурное решение: движок отдаёт **формула(детерминированные входы) → диапазон + флаг калибровки**, а не точку. Это и есть Risk #1, встроенный как фича (раздел 3).

Две дисциплины расчёта, которые нельзя путать:
- **Слой 1 — обмер чистой работы** (medição): сколько м²/м³/м элемента есть. Стандартизован относительно (но в PT нет обязательного нацстандарта — де-факто CSRMC/LNEC, ProNIC, добровольные).
- **Слой 2 — расход материала на единицу работы** (rendimento/consumo + waste%): kg/л/шт на единицу. **Стандартами обмера НЕ покрыт** — живёт в фичах производителей и сметных базах (CYPE). Obrato NormSet — это в основном Слой 2.

Институциональное признание Risk #1: и LNEC («корректирующая маржа»), и CYPE («carácter indicativo», «точный прогноз невозможен») сами называют свой выход калиброванной оценкой. Даже эталоны не претендуют на детерминизм Слоя 2.

## 1. По каждому материалу: метод, входы, диапазон, seed, источник, confidence

### 1.1 Плиточный клей (cola / tile adhesive)

**Метод/формула.** Расход НЕ из размера плитки. Цепочка: формат плитки + ровность основания → выбор зуба гладилки → толщина клеевого слоя → kg/m².

```
kg/m² = (kg/m²/мм порошка) × эффективная_толщина_мм × множитель_нанесения
  где эффективная_толщина ≈ зуб_мм / 2  (квадратный зуб)
                         ≈ зуб_мм / 3  (U-образный зуб)
        множитель = 1.0 (одинарное) | ≈1.5 (двойное / colagem dupla)
мешки = площадь × kg/m² × (1 + waste) / вес_мешка
```

Связка «зуб → слой = ½» (square) — устоявшееся правило из нескольких независимых источников (RUBI, Floor Elf): гребни клея при вдавливании плитки сминаются примерно вдвое.

**Входные переменные:** размер зуба гладилки (главный драйвер), плотность конкретного клея (kg/m²/мм), формат плитки (влияет косвенно — через выбор зуба и триггер двойного нанесения), одинарное/двойное нанесение, ровность основания, зона (пол/влажная/фасад → требуют 100% контакта).

**Диапазон:** 2–8 kg/m², центр ~4 для типового интерьера (зуб ~8 мм, одинарное). Реперы из фичей: Mapei Keraflex Maxi S1 = **1.2 kg/m²/мм**, плотность замеса 1500 kg/m³, 6–7 kg/m² гладилкой; Kerakoll H40 Gel ≈ **1.25 kg/m²/мм**, слой 2–15 мм; Weber col flex XL: талоча Nº6 — 4/6 (симплес/дупла), Nº9 — 5/7 kg/m². Мозаика зубом Nº4 ≈ 2 kg/m².

**Seed:** 3.5 [2–8] kg/m². Зуб 8 мм → слой ~4 мм → ~1.25×4 ≈ 5 (укладывается в диапазон).

**Источники:** [Mapei Keraflex Maxi S1 TDS](https://www.tilemountain.co.uk/pdf/Keraflex%20Maxi_S1_Datasheet.pdf), [Kerakoll H40 Gel PT](https://media.kerakoll.com/Sito_Web/techSheet/West_EU/Portogallo/3_Posa_ceramica/H40%20Gel_nuovo%202023_(pt).pdf), [Weber col flex XL FT](https://espacodearquitetura.com/wp-content/uploads/2021/05/FichaTecnica_webercolflexXL.pdf), [RUBI trowel guide](https://www.rubi.com/en/blog/trowel/), EN 12004/ISO 13007 (классификация C/D/R, 1/2, T/E/S1/S2 — **расход НЕ нормирует**).

**Confidence: средне-высокий.** Формула «зуб→слой→kg/m²» — устоявшаяся (несколько источников). Конкретные kg/m² по зубу — trade-practice разброс, согласованный между Mapei/Weber/Kerakoll, но без единого нормативного числа.

> **Verify-правки (важно):** мешок = **20 кг**, не 25 (Weber/Kerakoll/Mapei Maxi — все 20кг по своим фичам; прямо влияет на формулу мешков). Эвристика «6 мм→~3 kg/m²» **ошибочно приписана TAL** — TAL даёт толщину постели в мм, не kg/m²; переатрибутировать на калькуляторы. Триггер colagem dupla у Weber шире, чем «>900 см²»: **>900 см² ИЛИ наружу ИЛИ нестандартное основание**. BS 5385 verbatim говорит «solid bed для плитки ≥0.1 м²» + «50% недостаточно» — популярные 65%/80%/100% это trade/producer-интерпретация, не текст стандарта. Mapei TDS датирован 2015 — флаг «старый».

### 1.2 Затирка (betume de juntas / grout)

**Метод/формула.** Геометрический вывод: затирка заполняет объём швов на единицу площади.

```
kg/m² = ((A + B) / (A × B)) × C × D × k
  A = длина плитки (мм), B = ширина (мм), C = толщина плитки = глубина шва (мм),
  D = ширина шва (мм), k = плотностный коэффициент бренда
```

Член `(A+B)/(A×B)` — суммарная длина шва на единицу площади (1/мм): мельче плитка → больше. Это та же формула, что в калькуляторах TCNA/Laticrete/Omnicalculator.

**Входные переменные:** D — ширина шва (линейное, самое сильное влияние; **НЕ свободный выбор** — функция калибра партии, см. раздел 3), C — толщина плитки (= глубина шва), A/B — формат, k — плотность массы.

**Диапазон:** **0.1–2.9 kg/m²** (≈30× разброс только по геометрии). Центр ≈ 0.3 (типовой пол 300×300, шов 3 мм). Крупноформат 600×600 → ~0.1–0.2; мелкая плитка 100×100 при широком шве → до 2.9. Реперы плотности Mapei Ultracolor Plus: порошок 1400 kg/m³, замес **1980 kg/m³**, вода **20–26%**.

**Seed:** 0.3 [0.04–2.9] kg/m².

**Источники:** [Mapei Ultracolor Plus FT](https://www.obras360.pt/resources/fichastecnicas/0080050417mp/Ficha%20T%C3%A9cnica%20-%20Mapei%20Ultracolor%20Plus.pdf), [Omnicalculator Grout](https://www.omnicalculator.com/construction/grout), EN 13888 (CG/RG, CG1/CG2, A/W — **расход НЕ нормирует**), [ANSI A108.02 §4.3.8](https://ctasc.com/expert-answers/what-is-the-acceptable-tile-grout-joint-width-and-tile-lippage-for-tiles-with-one-edge-larger-than-15-inches/) (ширина шва).

**Confidence: высокий** на формулу и таблицу (40/40 ячеек воспроизводятся точно).

> **Verify-правка (критичная — research ошибся в своём же headline):** коэффициент k Mapei = **1.6**, НЕ ~1.7. Verify back-solved все 40 ячеек таблицы: **k=1.6 даёт RMSE 0.000 (40/40), k=1.7 — только 18/40**. Research выдвинул тезис «надо брать ~1.7, 1.6 занижает» — это **зеркально неверно**: 1.6 — точная калиброванная константа бренда (она сидит ниже wet-density 1980 намеренно, на недозаполнение/расшивку). Ловушка извлечения: формула и «1.6» в фиче — это **картинка** (pdftotext достаёт только легенду A/B/C/D; нужен pdftoppm). Для движка: k берётся из фичи конкретного бренда (≈1.6 Mapei), НЕ единый 1.6 для всех. Вода 20–26%, не 20–22%.

### 1.3 Гидроизоляция (impermeabilização / waterproofing)

**Метод/формула.** Один из немногих материалов с настоящей инженерной формулой.

```
кг/м² = число_слоёв × расход_на_слой
      = плотность_мокрой_смеси × сухая_толщина_плёнки (DFT)
Площадь = (Д×Ш пола) + (периметр_мокрой_зоны × высота_захода upstand) + примыкания
n_слоёв = required_DFT / max_на_слой   (число слоёв НЕ свободно)
```

Ключевая инженерная константа для **цементных обмазочных**: **2 кг/м² ≈ 1 мм мокрой плёнки** (Sika SikaTop Seal-107, Sika CemCrete, verbatim). Площадь гидроизоляции **больше** площади пола — на заход на стены.

**Входные переменные:** семейство (цементная обмазка vs жидкая PU/акрил мембрана), требуемый DFT (задаётся классом водной нагрузки DIN 18534), число слоёв (производное), плотность состава (back-solve дал реальный разброс **1.3–2.0 kg/m²/мм** между продуктами — продукт-специфика, не ошибка), пористость основания, высота захода (upstand), армирование углов.

**Диапазон:** цементная обмазка **3–4 kg/m², ≥2 слоя**. Реперы: weber.dry 824 «3–4.2 kg/m², мин 3, не более 1.5 kg/m² на слой» (Leroy PT); Isomat AQUAMAT (таблица нагрузка↔толщина — лучшая): влага 2 kg/m²~1.5мм, **вода без напора** 3 kg/m²~2мм, **под напором** 3.5–4.0 kg/m²~2.5мм; Kerakoll Aquastop Nanoflex ~1.2 kg/m²/слой; SikaTop Seal-107 ~4 kg/m² (2 слоя). Жидкие мембраны: ~1.5–2.5 kg/m², 2–3 слоя (подтверждённый верх ≈2.54, не 2.83).

**Seed:** 3.5 [3–4.2] kg/m² (≥2 слоя).

**Источники:** [SikaTop Seal-107 PDS](https://usa.sika.com/dam/dms/us01/d/pds-cpd-SikaTopSeal%20107-us.pdf), [Isomat AQUAMAT](https://www.isomat.eu/wp-content/uploads/2025/12/e-AQUAMAT.pdf), [weber.dry 824 FT](https://cdn.shopify.com/s/files/1/2316/0819/files/Ficha_Tecnica_weberdry_824.pdf?v=1651242503), [DIN 18534 (Dallmer)](https://www.dallmer.com/en/wissen/abdichtung-von-innenraeumen.php) (классы W0-I…W3-I), EN 14891 (классификация мембран).

**Confidence: высокий** на константу 2 kg/m²/мм и формулу (несколько Sika-источников verbatim).

> **Verify-правки:** AQUAMAT — «3 kg/m²~2мм» это **вода БЕЗ напора**, не «напор» (research сдвинул на строку; под напором — 3.5–4.0). DIN 18534: запрет акрила для W2-I, высота воды ≤10 см, общий заход 150–200мм вне душа — **NOT FOUND в процитированных URL** → помечать «требует верификации», не цитировать как факт. Старая цифра 2.83 kg/m² для Sikalastic-614 неверна (страница её больше не несёт).

### 1.4 Стяжка / наливной пол (betonilha / autonivelante)

**Метод/формула.** Линейность по толщине бесспорна. **Два РАЗНЫХ продукта, две разные единицы** — путать нельзя (разница реальна: полимерный сглаживатель vs цементно-песчаный балласт).

```
A. Наливной (autonivelante):  kg = площадь × толщина_мм × consumo[kg/м²/мм]
B. Балластная (betonilha):    kg = площадь × толщина_см × consumo[kg/м²/см]
```

**A. Autonivelante (kg/м²/мм):** Topeca Nível S **1.5** (слой 1–3 мм); Sika Sikafloor-100 Level **1.55** (плотность 1.9); Mapei Ultraplan Eco **1.6** (замес 2000); Ardex ARDITEX NA **1.65**; Weber weberfloor top **1.74** (1–10 мм). **Центр ~1.6, диапазон 1.5–1.74** (trade/рецептурный разброс между брендами).

**B. Betonilha (kg/м²/см):** Weber weberfloor flow **21** (3–10 см); weberfloor base **20** (4–8 см); традиционная 1:4 ≈ **20** (≈2000 kg/m³, max 4–5 см/слой). **Центр 20, диапазон 20–21.** Это ~10× единичного расхода автонивелира — разрыв реальный.

**Входные переменные:** толщина (= Risk #1, см. раздел 3), consumo продукта, для betonilha — соотношение цемент:песок (1:4 стандарт, 1:3…1:5 разброс).

**Seed:** наливная **1.7** (verify предлагает честнее **1.6**, диапазон 1.5–1.74) kg/м²/мм; балластная **~20** kg/м²/см.

**Источники:** [Topeca Nível S FT](https://topeca.pt/Imgs/produtos/004/26/24/ficheiros_pt/FT_-_Autonivelante_Nivel_S_rev09.pdf), [Sika Sikafloor-100 Level](https://prt.sika.com/dam/dms/pt01/w/sikafloor-100-leveles.pdf), [Mapei Ultraplan Eco](https://www.tradechoice.com/globalassets/productimages/103716_mapei-ultraplan-eco-3210-screed-technical-data-sheet.pdf), [Weber weberfloor flow (Adeo CDN)](https://media.adeo.com/media/2553289/media.pdf), [Eurobetonilhas](https://eurobetonilhas.com/index.php/betonilha), [BS 8204 SR-классы](https://www.speed-screed.com/knowledge-centre/surface-regularity-of-screed/).

**Confidence: высокий** на формулу и единицы; **средний** на центр (verify сдвинул).

> **Verify-правки:** Sika Sikafloor-100 = **1.55** (не 1.6), плотность **1.9** (не 1.8); Ardex = **1.65** (не 1.7); центр autonivelante честнее **1.6** (медиана пяти verbatim-точек), а не 1.7 — research завысил. BS 8204 SR-классы (SR1 ±3мм / SR2 ±5мм / SR3 ±10мм под 2-м рейкой) verbatim верны, но из speed-screed, не из gcs-concrete PDF (тот — скан без текста). kg цемента/песка для 1:4 — trade-practice оценка, не verbatim.

### 1.5 Краска (tinta) + грунт (primário)

**Метод/формула.** Устоявшаяся, физически выводимая.

```
Литры = (Площадь × число_слоёв) / rendimento[m²/L]
rendimento_теор = (volume_solids% × 10) / DFT_микрон   ← откуда бренд берёт цифру
Площадь_стен = Периметр × Высота − Σ(проёмы)
Площадь_потолка = Длина × Ширина
```

Заявленное в фиче rendimento — это уже **практическое** (prático), бренд срезал 10–20% с теоретического. Брать его, не теоретическое.

**Входные переменные:** впитываемость/пористость основания (главный фактор), шероховатость/текстура, тип краски и блеск, способ нанесения (валик/airless), разбавление (CIN: 1-й слой 10% воды, далее 5%), цветовой контраст/пигмент.

**Диапазоны (verbatim PT-фичи):** CIN Vinylmatt **11–15 m²/L/слой**, 2–3 слоя; Robbialac 015 — гладкие **10–15**, шероховатые **8–10**; Dyrup Dyruplad (это 2IN1!) **7–10** (гипсокартон). Грунт-селатор: CIN Cinolite Primer **6–8**, Cinolite HP **9–12**, Robbialac Hydroviplite **10–15**, Classidur Universal **9–11** m²/L, обычно **1 слой**.

**Число слоёв-стандарт:** краска default **2 слоя** (+1 при тёмном/насыщенном цвете); грунт **1 слой**.

**Seed:** краска **0.17 L/m²** (2 слоя, rendimento 12 m²/L [10–15]); грунт **0.10 L/m²** (1 слой, ~10 m²/L).

**Источники:** [CIN Vinylmatt FT](https://tintasepintura.pt/wp-content/assets/artigos/pt/ProdutosFichasTecnicas/ProdFichCin/10-250_VINYLMATT_P_.PDF), [Robbialac 015 FT](https://loja.tintasrobbialac.pt/pt/robbialac/downloads/documents_prods/015/Ficha%20Técnica.pdf), [Dyrup Dyruplad FT](https://dyrup.pt/kentico/getmedia/82adaf30-2bfd-405e-b283-b69bedb4cf4b/ficha-tecnica_Dyruplad.pdf), [CIN Cinolite Primer](https://cin.com/deco/pt/en/54850-cinolite-primer), [Promain — Theoretical Coverage](https://www.promain.co.uk/blog/theoretical-coverage-rates/), [ContractorMaths](https://contractormaths.com/paint-coverage-calculator/).

**Confidence: высокий** — формула бесспорна (коатинг-инженерия), rendimento verbatim из 7 фичей.

> **Verify-правки:** диапазоны по типам (стена 10–14, потолок 12–16) **GigaCalculator НЕ даёт** — снять как источник, оставить как trade-synthesis. Фраза «smooth surfaces under ideal conditions» — не verbatim, снять кавычки. Wastage 10–20%: полный диапазон на Promain, ContractorMaths = только 10%. Dyruplad — продукт 2IN1, не чистая финишная краска.

### 1.6 Геометрические нормы (без a-priori неопределённости в количестве)

- **Плитка:** количество = площадь / площадь_плитки, округлить вверх до коробок. Coverage **1 м²/м²**. Единственная мягкая переменная — waste% (см. 1.7).
- **Плинтус (rodapé):** длина = **периметр помещения** − проёмы дверей. Детерминированно.

### 1.7 Waste % (отход на подрезку и бой) — сквозная мягкая переменная

**Метод.** Отход — свойство **геометрии раскроя**, не плитки. Драйвер: отношение периметр/площадь + переиспользуемость обрезков (порог: обрезок <~50 мм трескается → в дело не идёт).

**Таблица (BS 5385 через MakeCalcs, verbatim-confirmed):**

| Паттерн | waste% |
|---|---|
| Straight (прямая) | **10%** |
| Brick / half-offset | 12% |
| Diagonal 45° | **15%** |
| Herringbone (ёлочка) | 18% |
| Basketweave | ~18% |

Адъюстменты (аддитивно): large format ≥600мм **+2%**; одно крупное препятствие +2%, L/U-форма +3%, несколько +4–5%; натуральный камень выше керамики (straight камень ~10% vs ~5% керамика).

**Seed:** **10% прямая / 15%+ диагональ** (industry-typical). Dye-lot: заказывать +10–15% из **одной партии** (повторный дозаказ даёт видимую разницу оттенка/калибра).

**Источники:** [MakeCalcs (BS 5385)](https://makecalcs.net/blog/tile-waste-factor-guide), [Atlas Tile](https://atlastile.com/determine-the-right-tile-quantity/), [Tile Choices — dye lots](https://tilechoices.com/blogs/tile-talk/understanding-dye-lots-in-tile-why-color-consistency-matters).

**Confidence: средний.** Иерархия (straight<brick<diagonal<herringbone) и порядок величин — устоявшиеся. Точное число внутри диапазона — **trade-practice разброс**, не норма. TCNA/NTCA единого публичного % НЕ публикуют.

> **Verify-правки:** «10%» — trade baseline, не цитата TCNA/NTCA. Сценарные числа (22–24% L-ванная, «мин 30% для LF», 60×60-в-5×8, perimeter-to-area) **ошибочно приписаны Surfaces Galore** — снять атрибуцию, понизить до «trade-practice, требует верификации». «5–10% attic stock» не подтверждён Tile Choices/Mann Tile. «Versailles ~20%» это Atlas, не BS 5385. Год BS 5385-3:2024 не подтверждён листингом BSI.

## 2. Что движок ОБЯЗАН захватывать (маппинг на NormAssumptions, гл.05)

Каждый вход ниже физически меняет расход и **должен быть полем NormAssumptions**, иначе движок угадывает.

| Вход (NormAssumption) | Тип | Влияет на | Источник значения | a-priori? |
|---|---|---|---|---|
| **Размер зуба гладилки** | enum (Nº4/6/9/12 мм) | клей kg/m² (×½ зуба) | выбор по формату плитки + основанию | НЕТ — выбор мастера |
| **Паттерн раскладки** | enum (straight/brick/diagonal/herringbone) | waste% (10→18%) | дизайн-решение | да (если задан) |
| **Толщина слоя** | число (мм/см) | стяжка, наливной, реборок, гидроизоляция | стяжка — Risk #1; гидро — класс DIN | стяжка НЕТ / гидро да |
| **Ровность основания** | enum/число (SR1/SR2/SR3 или мм/2м) | толщина стяжки, доп. клей, грунт | замер лазером/рейкой на объекте | **НЕТ — Risk #1** |
| **Число слоёв** | int | гидро, краска, стяжка | гидро = DFT/max_слой; краска default 2; грунт 1 | производное |
| **Формат плитки A×B×C** | мм×мм×мм | затирка (формула), waste, выбор зуба | datasheet плитки | да |
| **Ширина шва D** | мм | затирка (линейно) | ≥3× калибра партии (ANSI) | **НЕТ — Risk #1** |
| **Впитываемость / пористость** | enum (reboco/бетон/гипсокартон/окрашено/caiado) | rendimento краски (×~2), 1-й слой клея/гидро | осмотр + проба | **НЕТ — Risk #1** |
| **Множитель нанесения** | enum (одинарное/двойное) | клей (×1.5) | формат >900см² / влажная / фасад | да (по правилу) |

| **Соотношение цемент:песок** | enum (1:3…1:5) | betonilha масса | проект/прочность | да |

## 3. Что НЕ считается a priori → только калибровка (Risk #1)

Эти входы **физически не содержатся в Д×Ш×В** и не выводятся из габаритов. Для них движок ОБЯЗАН отдавать **диапазон + флаг «уточнить по факту / пробному участку»**, а не точку. Это не баг — это самая честная часть движка, и её прямо подтверждают LNEC и CYPE (раздел 0).

1. **Ровность/вертикальность основания → толщина стяжки (главный кейс).** Два визуально одинаковых помещения 20 м² могут требовать 5 мм и 45 мм средней толщины. Известно только после замера лазером/нивелиром. Практическая оценка: `средняя_толщина ≈ min_продукта + макс_перепад/2`; чем хуже планиметрия (SR3 vs SR1), тем больше скрытый объём. **Толщину запрашивать как калибруемый вход, не выводить.**

2. **Ширина шва D → расход затирки.** ANSI A108.02 §4.3.8: шов ≥ **3× фактического разброса калибра поставленной партии**, плюс коробление (warpage) самой длинной кромки. Известно только с плиткой в руках. До закупки — только диапазон (шов 2–10 мм → расход ×5–10).

3. **Впитываемость/пористость основания → расход краски, 1-го слоя клея и гидро.** Свежий reboco vs гипсокартон vs caiado меняют rendimento краски в разы (7 vs 15 m²/L). Все фичи прямо снимают с себя «substrate profile, porosity». Нужна проба первого слоя — лучший предиктор второго.

4. **Высота захода (upstand) и контур мокрой зоны → площадь гидроизоляции.** От 150 мм у раковины до 1800 мм в душе. Зависит от планировки (level-access? поддон? ванна?), не от Д×Ш×В пола. Площадь гидро > площади пола на величину, которую нельзя посчитать без разметки.

5. **Выбор зуба гладилки + техника мастера → расход клея.** Один кейс: мастер А кладёт зубом 8мм одинарно, мастер Б — 10мм двойным. Разброс легко ×1.5.

6. **Waste% → количество плитки.** Зависит от точки старта раскладки (центр vs край), дисциплины переиспользования обрезков, боя при резке, dye-lot. Не вычисляется из габаритов. Отдавать диапазон по паттерну + флаг калибровки по cut-list.

7. **Плотностный коэффициент k калиброван эмпирически, не теоретически.** Бренды ставят k ниже физической плотности (Mapei grout k=1.6 < wet-density 1.98 — намеренно, на потери). «Бери учебные 1.6/плотность замеса» промахивается на ±15%. **Брать k из фичи конкретного бренда.**

## 4. Честный итог: что твёрдое, что мягкое, что не достали, расхождения

### Устоявшиеся формулы (geometry + физика, надёжно)
- Затирка `(A+B)/(A×B)×C×D×k` — **40/40 ячеек таблицы Mapei воспроизводятся точно** при k=1.6.
- Краска `Литры = площадь × слои / rendimento` + теоретическая укрывистость `vol_solids×10/DFT` — коатинг-инженерия, бесспорно.
- Гидроизоляция `слои × плотность × DFT`, константа **2 kg/m²/мм** для цементной — несколько Sika-источников verbatim.
- Связка «зуб → слой ≈ ½» (square) / ⅓ (U) — несколько независимых источников.
- Линейность стяжки по толщине; количество плитки = площадь/площадь_плитки; плинтус = периметр.

### Trade-practice разброс (диапазон, не точка)
- **waste%** внутри паттерна (diagonal 15–25%, herringbone 15–20%) — иерархия твёрдая, точное число нет. TCNA/NTCA публичного % не дают.
- **kg/m² клея по зубу** — согласован между Mapei/Weber/Kerakoll, но без нормативного числа (3–8, центр 4).
- **Центр consumo autonivelante** ~1.6 [1.5–1.74] — рецептурный разброс брендов.
- **coverage% 65/80/100** — producer-интерпретация, не текст BS 5385 («≥0.1м² → solid bed», «50% недостаточно»).
- **Соотношение betonilha 1:4** (1:3…1:5), «не вычитать проёмы <0.5м²», wastage 10–20% — trade-эвристики.

### Не достали / требует верификации
- **leroymerlin.pt бот-блокнут (403)** → обойдено фичами CIN/Weber/Mapei/Kerakoll/Sika напрямую.
- **LNEC Fichas de Rendimento** (числа) — бот-блок livraria/repositorio.lnec.pt; зеркала Scribd не verbatim-подтверждены.
- **DIN 18534:** запрет акрила для W2-I, вода ≤10см, заход 150–200мм вне душа — NOT FOUND в процитированном Dallmer URL → «требует верификации».
- **BS 5385-3:2024** — год не подтверждён листингом BSI. Платный → проценты только через вторичные источники.
- **kg цемента/песка для betonilha 1:4** — trade-оценка, не verbatim.
- **Caderno Técnico Mapei betonilhas** — URL правдоподобен, дословно не открыт.

### Расхождения, которые verify исправил относительно research (ключевая ценность прохода)
1. **Grout k = 1.6, НЕ ~1.7** — research выдвинул зеркально неверный headline-тезис; verify back-solved 40/40 ячеек. **Самая важная правка.**
2. **Мешок клея = 20 кг, НЕ 25** — прямо ломал формулу мешков.
3. **Autonivelante центр 1.6, НЕ 1.7**; Sika 1.55/плотность 1.9 (не 1.6/1.8); Ardex 1.65 (не 1.7).
4. **AQUAMAT:** «3 kg/m²» = вода БЕЗ напора (research сдвинул на строку; под напором 3.5–4.0).
5. **Атрибуции:** TAL не даёт kg/m² (даёт мм); Surfaces Galore не даёт сценарных waste-чисел; GigaCalculator не даёт диапазонов по типам краски. Все три — переатрибутировать или понизить до «trade-practice».
6. **Sikalastic-614:** верх 2.54 kg/m², не 2.83.

### Один сквозной вывод для архитектуры движка
Для **каждого** материала: `consumo/k/rendimento` берётся из фичи **конкретного бренда** (не единая зашитая константа), площадь/формат — детерминированно, а **толщина-по-кривизне, ширина-шва-по-калибру, впитываемость, upstand, waste%, выбор зуба** — это поля NormAssumptions с режимом «диапазон по умолчанию → калибровка по пробному участку/первому слою». Самый честный UX: оценка-диапазон до калибровки → уточнение по факту первого слоя/раскладки. Это и есть Risk #1, превращённый из бага в фичу.
